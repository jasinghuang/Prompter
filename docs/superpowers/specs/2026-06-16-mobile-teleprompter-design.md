# 手机提词器 设计文档

- **日期：** 2026-06-16
- **状态：** 已与用户确认，待编写实现计划
- **参考实现：** `ref_code/`（`source_package/` 为 UI/功能核心参考，复刻其功能与观感，不复刻其技术栈）

## 1. 项目概述

一个面向**短视频 / Vlog 拍摄**场景的手机提词器。

核心价值：拍摄者把手机/iPad 横放架在三脚架上，对着镜头念稿，屏幕上大字号逐字滚动、当前字高亮停在视线中线上，省去背稿。聚焦核心提词体验，不做语音同步、不做后端。

**最终产物：** 一个单 HTML 文件，用浏览器（Chrome 等）本地打开即用，无需联网、无需服务器、无需安装。

## 3. 技术架构

### 3.1 技术栈

- **Vite + React 18 + TypeScript** —— 开发体验、类型安全
- **TailwindCSS** —— 与 ref_code 一致的原子化样式（黑底 + 黄强调 + 玻璃态）
- **vite-plugin-singlefile** —— 构建时把 JS/CSS 全部内联进单个 `index.html`，无外部资源依赖
- **纯前端、零后端**

### 3.3 数据模型

```ts
interface Script {
  id: string;          // crypto.randomUUID()
  title: string;
  content: string;
  createdAt: number;
  updatedAt: number;
}

interface TeleprompterSettings {
  fontSize: number;        // 字号 px，默认 64，范围 24–120
  letterSpacing: number;   // 字间距 em，默认 0.05，范围 0–0.3
  lineHeight: number;      // 行高倍数，默认 1.6，范围 1.0–2.5
  scrollSpeed: number;     // 滚动速度 WPN（字/分钟），默认 160，范围 60–360
  mirror: boolean;         // 镜像翻转，默认 false
}
```

### 3.4 视图状态机

`App.tsx` 维护一个视图状态：`'list' | 'prompter' | 'editor'`，配合 `activeScriptId`：

- `list` → 稿件管理首页
- `prompter` → 提词器主视图（核心）
- `editor` → 编辑文案

### 3.5 数据流

`localStorage` ↔ `useStore` hook ↔ React 组件。

- 两把 key：`prompter_scripts`（稿件数组）、`prompter_settings`（全局设置）
- 稿件数据按篇存储；提词设置全局共享
- 容错：读取时 JSON 解析失败则回退空列表 / 默认设置，不崩溃

## 4. 提词器核心机制（最重要）

### 4.1 逐字渲染与三态高亮

稿件按字符拆成 token 数组，每个字符渲染为一个 `<span>`（换行符渲染为 `<br>`）。维护 `currentIndex`（当前读到第几个字符），据此渲染三类状态：

- **已读字** → `neutral-600`（暗灰）
- **当前字** → 黄色（`#eab308`）+ 略放大（`scale(1.08)`）
- **未读字** → `neutral-400`（亮灰）

### 4.2 滚动跟随

- `currentIndex` 推进时，计算该字符元素的目标位置
- 用 `requestAnimationFrame` 把 `scrollTop` 朝目标位置**缓动过渡**（不瞬移），手感丝滑且可控
- 改进点：不用 ref_code 的 `scrollIntoView` 内置动画（快速滚动时卡顿堆积），改用独立平滑跟随
- 屏幕中线画一条**阅读线**（黄色横线，标 "Reading Area"），当前字始终停在这条线上

### 4.3 滚动速度

单位 **WPN（字/分钟）**，符合中文直觉，与 ref_code 一致：

- 预设档：龟速 80 / 标准 160 / 快语 240 / 极速 320
- 连续滑块：60–360，可微调
- 计算：`msPerChar = 60000 / WPN`，按帧累积推进 `currentIndex`

### 4.4 两种操控模式

| | 自动模式 | 手动模式 |
|---|---|---|
| 推进 | 按 WPN 匀速自动前进 | 不自动前进 |
| 控制方式 | 播放/暂停按钮 + 快进后退（±20 字） | ① 点任意字跳转 ② 手指上下拖拽自由滚动 |
| 高亮跟随 | 跟随自动推进 | 拖到哪里，高亮离阅读线最近的字 |

### 4.5 边界处理

- 滚到结尾自动停止
- 播放中可随时暂停 / 切手动
- 切换模式不重置阅读位置

## 5. 界面设计

### 5.1 提词器主视图（横屏）

横放设备后应用铺满横屏。布局（自上而下）：

1. **顶部栏**（半透明渐变，不挡文字）：返回 ← · 录制计时器（红点 + `00:00`）· 稿件标题 · 编辑文案 ✎ · 设置 ⚙
2. **进度条**（顶部栏下方一条细线）：左端 = 已用时长，右端 = 预估总时长，中间填充条 + 圆点把手；点/拖可跳转任意位置
3. **阅读区**：大字号逐字高亮文字，水平限宽、居中、左右扫读
4. **阅读线**：屏幕中线黄色横线
5. **底部胶囊控制条**（居中浮动，半透明玻璃态）：模式切换（🖱 手动 / ⏱ 自动）· 播放控制（⏮ 退 / ▶ 播放 / ⏭ 进）· 速度档（当前 WPN，**仅在自动模式显示**——手动模式无自动推进速度，不显示）

- 上下留白约 `16vh / 18vh`，保证首尾字符滚动时有缓冲
- 所有控件半透明浮层，录制时不抢画面

视觉风格：黑底（`#050505`）+ 黄色强调（`#eab308`）+ 中性灰 + 玻璃态（`backdrop-blur`）+ 胶囊形，沿用 ref_code。

### 5.2 计时器

- **实际录制时长**——从点播放开始累加的墙钟时间，暂停即停
- 等宽字体显示（`font-variant-numeric: tabular-nums`），格式 `mm:ss`

### 5.3 进度条

- 左端：已用时长（与计时器同源）
- 右端：**预估总时长** = 稿件总字数 ÷ 当前 WPN，切速度时跟随变化，用于预判"这稿大约念多久"
- 可点/拖跳转任意位置（跳转后 `currentIndex` 同步更新）；**自动模式**下从新位置继续自动推进，**手动模式**下停在新位置

### 5.4 设置面板（右侧抽屉）

点顶部 ⚙ 从右侧滑出。设置**实时生效**（拖动即变，所见即所得）。**不内嵌编辑文案的 textarea**（编辑独立到专门视图）。

| 调节项 | 控件 | 范围 | 默认 |
|--------|------|------|------|
| 字号 | 滑块 | 24–120 px | 64 px |
| 字间距 | 滑块 | 0–0.3 em | 0.05 em |
| 行距 | 滑块 | 1.0–2.5 倍 | 1.6 倍 |
| 滚动速度 | 滑块 | 60–360 WPN | 160 WPN |
| 镜像翻转 | 开关 | 开/关 | 关 |

**镜像翻转：** 对整个内容容器做 `scaleX(-1)`（水平翻转），用于分光镜反射玻璃。注意翻转后文字左右反过来，正常肉眼无法阅读，是分光镜专用。默认关闭。

## 6. 稿件管理与编辑

### 6.1 稿件列表（首页）

- 卡片式列表：手机竖屏单列；iPad / 横屏可两列
- 每张卡片：标题 + 正文预览（截断 2–3 行）+ 更新时间
- 卡片点击 → 进入提词器
- 卡片右上角小按钮 → 编辑 / 删除（删除二次确认弹窗）
- 顶部：搜索框（按标题 + 正文模糊匹配）+ 新建按钮
- 空状态引导（"新建第一篇稿件"）

### 6.2 编辑文案

- 入口：① 列表卡片上的"编辑"按钮 ② 提词器顶部栏 ✎（边提词边改）
- 纯文本编辑：标题输入框 + 正文大文本域
- **自动保存**：输入即存（防抖 ~500ms 后写 `localStorage`），无保存按钮
- 从提词器 ✎ 进入编辑、改完返回：
  - 提词器自动加载最新内容
  - 若改动**未跨越**当前阅读位置 → 保持原位置
  - 若改动**跨越**当前阅读位置 → **重置到开头 + 轻提示**

## 7. 移动端体验增强

| 项 | file:// 下可用性 | 说明 |
|---|---|---|
| ① 屏幕常亮 Wake Lock | ✅ Chrome(Mac/Win/Android) · ⚠️ iOS 不可靠 | 检测不可用时降级为"提示用户调长系统自动锁屏" |
| ② 全屏模式 | ✅ | Fullscreen API，Chrome 的 `file://` 下可用 |
| ③ 横屏自适应 | ✅ | 纯 CSS/JS 响应 `orientation`；竖屏进提词器时轻提示"请横放设备" |
| ④ 触控优化 | ✅ | 播放/暂停按钮大易点；手动模式拖拽用 `touch` 事件 + 惯性；防误触 |
| ⑤ 防误退出 | ✅ 桌面 | `beforeunload`，移动端受限 |

**不包含：** ~~Service Worker 离线缓存~~ —— 纯本地文件天然离线，不需要。

**iOS 防息屏降级：** Wake Lock API 要求安全上下文，`file://` 在 iOS（Safari 及 iOS Chrome，底层均为 WebKit）下基本不工作。检测到不可用时，界面给出轻提示："当前设备无法自动保持常亮，请到系统设置把自动锁屏调长"。其余平台（Chrome on Mac/Win/Android）正常使用 Wake Lock。

## 9. 不做的事（YAGNI）

明确排除，避免范围蔓延：

- 语音同步（飞书 ASR / 本地 ASR / 语音对齐引擎）
- 后端、服务器、API 代理
- 跨设备稿件同步
- 用户账号 / 登录
- Service Worker / PWA 安装
- 强制旋转屏幕（iOS 不可行，靠提示 + 自适应）
- 主题切换 / 多套配色（保持 ref_code 黑底黄强调单一风格）
- 富文本编辑（纯文本即可）
- 导入/导出稿件文件（首期不做，多稿存 localStorage）

## 10. 测试策略

- **滚动与高亮：** 自动模式按给定 WPN 推进，断言 `currentIndex` 随时间正确递增、到结尾停止
- **滚动跟随：** `currentIndex` 变化后，目标字符元素最终滚动到中线附近（容差断言）
- **手动模式：** 拖拽后高亮跟随离中线最近的字符
- **设置实时生效：** 修改字号/速度/镜像后，DOM 样式与 store 状态一致
- **持久化：** 稿件与设置写入 `localStorage`，刷新后可恢复；解析失败回退默认值不崩溃
- **自动保存：** 编辑输入后防抖窗口外，`localStorage` 已更新
- **编辑返回定位：** 改动未跨位置保持、跨越则重置到开头
- **进度/计时：** 计时器随播放累加、暂停停止；预估总时长 = 字数 ÷ WPN
- 手动/E2E 验证：Chrome（Mac/Android）Wake Lock 与全屏；iOS 降级提示路径

## 11. v2 优化（2026-06-18）

基于实际使用反馈，对提词器做以下优化与重构（覆盖上方 v1 的相关条款，以本节为准）：

### 11.1 滚动：丝滑无极（重构）

- **问题：** v1 用「逐字推进 currentIndex → 缓动 scrollTop 到该字」。currentIndex 离散跳变导致一顿一顿；且 scrollTop 被浏览器量化为整数像素，低速（每帧 < 1px）时累积跳跃，卡顿明显。
- **新方案：** 改为 **CSS `transform: translate3d(0, -offset, 0)`** 驱动内容位移，走 GPU 合成层、支持亚像素，任何速度（含 30 WPN 极慢）都丝滑。
- **滚动模型反转：** 滚动位移 `offset` 为源头（`useAutoScroll` 用 rAF 按 `pxPerSec` 连续累加 `offset` 并写 transform）；高亮位置由 `offset` 反推（中线对应的字）。不再有"逐字推进 + 缓动"。
- **像素速度换算：** `pxPerSec = (maxOffset ÷ 可读字数) × (WPN ÷ 60)`。
- 容器结构：`viewport(overflow hidden)` + `content(transform 位移, position relative, willChange transform)`。

### 11.2 逐行高亮（取代逐字）

- 由逐字改为**逐视觉折行行**高亮：纯函数 `computeLineIndices(offsetTops)` 把相同 `offsetTop`（容差 1px）归为同一行；当前字所在行整行高亮（黄字加粗），其上为已读（暗）、其下为未读（亮）。
- 测量在 `ScriptText` 的 `useLayoutEffect` 中完成，依赖 `layoutKey`（字号/字间距/行距/对齐/内边距/宽度）变化时重测；tokenize 用 `useMemo` 缓存。

### 11.3 阅读区色块

- v1 的 1px 中线改为 **2 行高的淡黄色带**（`rgba(234,179,8,0.07)` 底 + 上下淡边线），当前行落在带内，提供稳定注视区。

### 11.4 设置项扩展

`TeleprompterSettings` 新增：
- `horizontalPadding: number`（左右内边距 %，4–20，默认 8）
- `textAlign: 'left' | 'center' | 'right'`（默认 **left**）

设置面板新增「两边间距」滑块与「对齐方式」三按钮。`loadSettings` 合并默认值的容错使旧数据自动补齐新字段。

### 11.5 速度：直调 + 扩范围

- 自动模式下，**控制条内置速度滑块**（无需进设置即可调）。
- 范围由 60–360 扩至 **30–600 WPN**（`SPEED_MIN/MAX`）。
- 快捷键：**空格** 播放/暂停、**↑/↓** 调速度（步进 20，Shift 微调 5）。

### 11.6 交互统一

- 手动浏览：**统一 pointer 事件**实现拖拽自由滚动（区分 tap/drag，6px 阈值）。
- **双击 / 双指轻触**（300ms 内两次未拖拽）→ 播放/暂停。
- 点击按钮后自动失焦，避免空格被按钮拦截或双重 toggle。

### 11.7 稿件管理增强

- **一键清空全部**（`clearAll`，二次确认）。
- **导入稿件**（`importScript`）：导入面板支持 ① 粘贴文本 ② 选择 `.txt/.md` 文件（多选，每个文件建一篇）。用于从苹果备忘录导入（备忘录全选复制粘贴，或「发送副本→存储到文件」导出 txt）。
- > 注：v1 第 9 节"导入/导出稿件文件（首期不做）"在 v2 调整为「导入」已实现；导出仍不做。

### 11.8 测试

- 新增 `lib/lines.test.ts`（computeLineIndices）、`hooks/useAutoScroll.test.ts`（transform 位移、亚像素、到顶到底）。
- 全量 94 测试通过；逐行/滚动的真实视觉行为靠浏览器手动验证（jsdom 无布局）。
